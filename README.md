# Motobro — бот для мото-группы

Телеграм-бот на **Node.js 20+**, **TypeScript**, **Telegraf**, **Express** (webhook), **PostgreSQL**, **Prisma**. Помогает участникам видеть, **кто сейчас катается** и **где примерно**, без постоянного GPS-трекинга: используются только **live location** и **ручной старт/стоп поездки**.

## Что умеет бот

- `/start` — коротко объясняет идею и экономию батареи  
- `/help` — список команд  
- `/ride` — выбор длительности (15–120 мин) и инструкция отправить **live location**  
- `/stop` — завершить поездку  
- `/who` — кто сейчас в активной поездке (примерные районы, время с последнего обновления)  
- `/where` — те же участники, акцент на районах (и ссылка на карту в браузере)  
- `/status` — твой статус  
- `/privacy` — что хранится и зачем  
- `/settings` — флаги на будущее, удаление последней точки  
- `/groupsettings` — для админов: таймаут неактивности, объявления  

Опционально в группу уходит короткое сообщение вроде: *«Имя выехал. Примерный район: …»* (если включено в настройках группы).

## Ограничения (так задумано)

- Карта — по ссылке в браузере (`/where`, `/who`), в самом Telegram только текст; нет истории маршрута  
- Нет точных координат в групповых списках по умолчанию — только **примерный район**  
- Нет «тяжёлого» геокодинга: при ошибке или отключении — подпись по округлённым координатам  
- v1 без сложных уведомлений и без отображения точного трека  

## Почему не жрёт батарею

- Нет своего фонового трекера — только то, что присылает **Telegram** (live location по твоей команде)  
- На сервере хранится **последняя точка** и **метка района**, не история движения  
- Обновления обрабатываются **только когда приходит апдейт от Telegram**  

## Требования

- Node.js **20+**  
- PostgreSQL **14+** (удобно **Render Postgres**)  

## Локальный запуск

1. Склонируй репозиторий и установи зависимости:

```bash
npm install
```

2. Скопируй `.env.example` в `.env` и заполни переменные (см. ниже).

3. Примени миграции:

```bash
npm run prisma:migrate
```

4. Запуск в dev (нужен публичный HTTPS для webhook; для отладки можно временно использовать **ngrok** и подставить `WEBHOOK_URL`):

```bash
npm run dev
```

5. Сборка и прод:

```bash
npm run build
npm start
```

Переменные окружения см. в `.env.example`. **Секрет webhook** должен совпадать с тем, что передаётся в `setWebhook` (`secret_token`).

## Деплой на Render (Free)

1. Создай **PostgreSQL** на Render (Free), скопируй **Internal/External Database URL** в `DATABASE_URL` веб-сервиса.  
2. Создай **Web Service**, подключи этот репозиторий.  
3. **Build Command** (пример):

```bash
npm ci && npm run build && npx prisma migrate deploy
```

4. **Start Command**: `npm start`  
5. В **Environment** задай минимум:
   - `BOT_TOKEN` — от BotFather  
   - `TELEGRAM_WEBHOOK_SECRET` — длинная случайная строка  
   - `DATABASE_URL` — из Render Postgres (часто подставляется через связь с БД)  
   - `WEBHOOK_URL` — можно **не задавать**: приложение использует **`RENDER_EXTERNAL_URL`**, который Render выставляет сам (или укажи вручную `https://<твой-сервис>.onrender.com`)  
6. После деплоя бот при старте вызывает `setWebhook` и `setMyCommands`.  

Файл `render.yaml` — ориентир для Blueprint; при необходимости поправь имена сервиса и БД.

**Health check:** `GET /health` → `{ "ok": true }`  

**Webhook:** `POST /telegram/webhook` (Telegraf + проверка `X-Telegram-Bot-Api-Secret-Token`).

## Как создать бота и получить BOT_TOKEN

1. В Telegram открой **@BotFather** → `/newbot` → имя и username.  
2. Скопируй **токен** — это `BOT_TOKEN`.  

## Как задать секрет webhook

Придумай строку **не короче 16 символов** (случайные буквы/цифры). Это `TELEGRAM_WEBHOOK_SECRET`. Тот же секрет Telegram будет слать в заголовке; бот его проверяет.

## Как добавить бота в группу

1. Добавь бота в группу как обычного участника.  
2. Дай право **отправлять сообщения** (для объявлений о поездках).  
3. Команды вроде `/who` и `/groupsettings` вызывай **в группе**; `/ride` удобнее в **личке** с ботом.  

## Какие команды доступны

См. `/help` в боте — список синхронизирован с меню команд Telegram.

## Какие данные хранятся

- Профиль Telegram (id, имя, username — как в Telegram)  
- **Последняя** геоточка и **примерное название района** (или запасная подпись по координатам)  
- Сессии поездок: статус, время, последняя точка/район, привязка к группе (если есть)  
- Кэш геокодинга по округлённым координатам  
- Настройки пользователя и группы  

Полный маршрут **не хранится**.

## Что делать, если не приходит live location

- Проверь, что в Telegram выбран именно **«Трансляция» / live location**, а не одноразовая метка.  
- Убедись, что выбрал длительность в `/ride` и отправил live location **в чат с ботом** (или там, где бот может читать апдейты).  
- На iOS/Android иногда мешает режим энергосбережения — кратко открой карту и заново включи трансляцию.  

## Как остановить поездку

- Команда `/stop` в личке или в группе  
- Если долго нет обновлений live location, поездка станет **истёкшей** по таймауту неактивности (по умолчанию **15 минут**, настраивается для группы)  

## Переменные окружения

См. `.env.example`. Кратко:

| Переменная | Назначение |
|------------|------------|
| `NODE_ENV` | `development` / `production` |
| `PORT` | Порт HTTP (Render подставляет сам) |
| `BOT_TOKEN` | Токен бота |
| `DATABASE_URL` | PostgreSQL connection string |
| `WEBHOOK_URL` | Базовый URL сервиса (без слэша в конце); на Render можно опустить, если есть `RENDER_EXTERNAL_URL` |
| `TELEGRAM_WEBHOOK_SECRET` | Секрет для webhook |
| `DISABLE_REVERSE_GEOCODING` | `true` — не дергать Nominatim |
| `NOMINATIM_BASE_URL` | Базовый URL Nominatim |
| `LOG_LEVEL` | Уровень логов Pino |
